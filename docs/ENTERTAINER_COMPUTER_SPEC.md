# エンターテイナー＝コンピューター スペック定義

エンターテイナーを**ブラックボックス**としてのプロセッサーとメモリを持つ計算資源と定義し、客観的な In/Out とスペックを記述する。

---

## 1. 概念モデル

```
                    ┌─────────────────────────────────────┐
                    │     エンターテイナー（ブラックボックス）    │
   Input ──────────►│  ┌─────────────┐  ┌─────────────┐   │──────────► Output
                    │  │  Processor  │  │   Memory    │   │
                    │  │  （処理装置）  │  │  （記憶装置）  │   │
                    │  └─────────────┘  └─────────────┘   │
                    └─────────────────────────────────────┘
```

- **Processor**: 入力を受け取り、リアルタイムでパフォーマンスを生成する処理装置
- **Memory**: 過去の経験・学習・パターンを保持し、Processor が参照する記憶装置
- **Input**: 客観的に観測可能な入力
- **Output**: 客観的に観測可能な出力

---

## 2. In/Out の記述ルール

- **客観的**: 第三者によって観測・記録・検証可能なものに限定
- **観測可能**: 映像・音声・記録・証言などで裏付け可能
- **推測・解釈は含めない**: 「意図」「感情」は In/Out に含めず、観測可能な現象のみ

## 4. In/Out の集約と共通軸

上記を集約し、エンターテイナー横断で比較可能な軸に整理する。

| 軸 | 説明 | 該当 In/Out |
|----|------|-------------|
| **コンテンツ入力** | 楽曲・脚本・ルールなど、パフォーマンスの「材料」 | 楽曲、脚本、対戦相手、ストーリー |
| **環境入力** | 会場、機材、照明、観客 | 会場、機材、観客反応 |
| **人的入力** | 共演者、監督、相手 | 共演者、メンバー、対戦相手 |
| **時間入力** | 制約、スケジュール | 時間制約 |
| **身体出力** | 声、動き、表情 | 歌唱、演技、ダンス、技 |
| **集団出力** | 複数人での統合出力 | グループ演奏、試合の流れ |
| **言語出力** | セリフ、トーク、マイク | ナラティブ、マイクパフォーマンス |

## 5. 計算資源としてのスペック定義

In/Out の集約結果を元に、客観的な計算資源としての能力スペックを定義する。

### 5.1 Processor スペック

| スペック項目 | 説明 | 測定・評価方法 |
|--------------|------|----------------|
| **入力帯域** | 同時に処理できる入力の種類・量 | In の種類数、複雑度 |
| **リアルタイム性** | 入力に対する応答遅延 | 観客反応への応答速度、即興度 |
| **並列処理** | 複数チャネルの同時処理 | 歌唱＋ダンス、演技＋動き など |
| **適応力** | 想定外入力への対応 | ハプニング対応、即興 |
| **出力安定性** | 再現性、一貫性 | 複数公演での出力のばらつき |
| **ピーク処理能力** | 最大負荷時の性能 | 大規模ライブ、長時間公演 |

### 5.2 Memory スペック

| スペック項目 | 説明 | 測定・評価方法 |
|--------------|------|----------------|
| **保持容量** | 保持できるパターン・経験の量 | レパートリー数、役の数 |
| **保持期間** | 記憶の持続時間 | 長年保持する楽曲、振付 |
| **想起速度** | 必要な情報の取り出し速度 | 即座にパフォーマンスに反映 |
| **参照精度** | 正確に再現できる度合い | 振付・セリフの再現精度 |
| **転移学習** | 他領域への応用 | ドラマ→バラエティ、楽曲→楽曲 |

### 5.3 スペック記述フォーマット

各エンターテイナーごとに、以下のフォーマットでスペックを記述する。

```yaml
entertainer: [名前]
genre: [ジャンル]
unit: individual | group

processor:
  input_bandwidth: [高|中|低]  # 入力の種類・量
  realtime: [高|中|低]        # リアルタイム応答
  parallel: [高|中|低]        # 並列処理能力
  adaptation: [高|中|低]      # 適応力
  stability: [高|中|低]       # 出力安定性
  peak_load: [高|中|低]       # ピーク処理能力

memory:
  capacity: [高|中|低]        # 保持容量
  retention: [高|中|低]       # 保持期間
  recall_speed: [高|中|低]   # 想起速度
  precision: [高|中|低]      # 参照精度
  transfer: [高|中|低]       # 転移学習

notes: [特記事項]
```

## 7. 今後の更新方針

1. **In/Out の追加・修正**: 新たな観測可能な入出力があれば追記
2. **スペックの精緻化**: 定量的な指標の導入を検討
3. **比較分析**: スペック間の相関、ジャンル横断の傾向を分析
4. **検証**: 一次資料（映像、記録）に基づくスペックの検証
